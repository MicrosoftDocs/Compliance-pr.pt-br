<span data-ttu-id="8c569-114">O Exchange Online, o Microsoft Teams, o SharePoint Online e o OneDrive for Business também usam a Criptografia de Serviço na camada de aplicativos para criptografar o conteúdo do cliente.</span><span class="sxs-lookup"><span data-stu-id="8c569-114">Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business also use Service Encryption at the application layer to encrypt customer content.</span></span> <span data-ttu-id="8c569-115">A Criptografia de Serviço fornece recursos de proteção e gerenciamento de direitos sobre a forte proteção de criptografia.</span><span class="sxs-lookup"><span data-stu-id="8c569-115">Service Encryption provides rights protection and management features on top of strong encryption protection.</span></span> <span data-ttu-id="8c569-116">Ele também permite a separação entre sistemas operacionais Windows e os dados do cliente armazenados ou processados por esses sistemas operacionais.</span><span class="sxs-lookup"><span data-stu-id="8c569-116">It also allows for separation between Windows operating systems and the customer data stored or processed by those operating systems.</span></span>

<span data-ttu-id="8c569-118">Os produtos e serviços do Microsoft 365 usam protocolos de transporte fortes, como TLS, para impedir que terceiros não autorizados a espionagem dados do cliente enquanto ele se move pela rede.</span><span class="sxs-lookup"><span data-stu-id="8c569-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="8c569-119">Exemplos de dados em trânsito incluem mensagens de email que estão em processo de entrega, conversas ocorrendo em uma reunião online ou arquivos replicados entre datacenters.</span><span class="sxs-lookup"><span data-stu-id="8c569-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="8c569-122">Como o Microsoft 365 gerencia as chaves usadas para criptografia?</span><span class="sxs-lookup"><span data-stu-id="8c569-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="8c569-130">Os clientes com requisitos para controlar suas próprias chaves de criptografia raiz podem aproveitar a Criptografia de Serviço com a Chave do Cliente.</span><span class="sxs-lookup"><span data-stu-id="8c569-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="8c569-131">Usando a Chave do Cliente, os clientes podem gerar suas próprias chaves criptográficas usando um HSM (Módulo de Serviço de Hardware) local ou AZure Key Vault (AKV).</span><span class="sxs-lookup"><span data-stu-id="8c569-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="8c569-132">As chaves raiz do cliente são armazenadas em AKV, onde podem ser usadas como raiz de um dos chaves que criptografam dados ou arquivos da caixa de correio do cliente.</span><span class="sxs-lookup"><span data-stu-id="8c569-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="8c569-133">As chaves raiz do cliente só podem ser acessadas indiretamente pelo código de serviço do Microsoft 365 para criptografia de dados e não podem ser acessadas diretamente por funcionários da Microsoft.</span><span class="sxs-lookup"><span data-stu-id="8c569-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
